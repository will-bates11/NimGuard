Notes:
  - 0x7FF682751000 is the base load address of notepad.exe plus 0x1000,
    which lands at the start of the .text section.
  - Two NOP bytes (0x90 0x90) are written via WriteProcessMemory. This is a
    benign test: the first two bytes of the .text section are overwritten, but
    the instruction boundary is preserved because we write exactly two bytes
    aligned to the section start.
  - In a real hardening workflow you would target a specific call site returned
    by --analyze or --disasm, not an arbitrary offset.
  - ASLR is active: the base address will differ between runs. Use the
    powershell command above (or tasklist /m) to locate the current base.
