Notes:
  - Eight NOP bytes (0x90 * 8) are injected at the start of the .text section.
    A full 8-byte word is written to avoid a read-before-write PTRACE_PEEKDATA
    step; on Linux kernels with strict ptrace hardening, PEEKDATA on r-xp pages
    can fail even after PTRACE_ATTACH, so aligning the write to a word boundary
    ensures a clean PTRACE_POKEDATA without a preceding read.
  - In a real hardening scenario you would target a specific call site returned
    by --analyze or --disasm rather than the section base address.
  - WSL2 ptrace_scope=1: /tmp/traceable_sleep calls prctl(PR_SET_DUMPABLE, 1)
    to allow attachment. Standard /usr/bin/sleep does not; use traceable_sleep
    or launch the target as a child of nimguard via PTRACE_TRACEME.
  - ASLR is active: BASE will differ between runs. Always read /proc/<pid>/maps
    at attach time to locate the current .text base.

Platform: Ubuntu 24.04 WSL2, nimguard_linux x64
